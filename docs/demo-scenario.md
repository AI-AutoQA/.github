# 데모 시나리오

AutoQA 데모 진행 순서입니다. 사전에 `docker compose up --build`로 전체 서비스를 기동해 두세요.

---

## 사전 준비

```bash
cd infra
cp .env.example .env
docker compose up --build
```

브라우저에서 http://localhost:3000 접속을 확인합니다.

---

## 시나리오 1 — Smoke 테스트 자동 실행

**목표**: URL만 입력하면 AI가 smoke 스크립트를 생성하고 자동으로 테스트가 실행됨을 보여줍니다.

1. 사이드바에서 **대시보드** 클릭
2. **+ 프로젝트 추가** 버튼 클릭
3. 아래 값 입력 후 **생성**:
   - 이름: `데모 프로젝트`
   - URL: `https://www.google.com`
4. 생성된 카드 클릭 → 프로젝트 상세 페이지 이동
5. **+ 테스트 실행** 버튼 클릭
6. 테스트 유형: `Smoke - 기본 동작 확인` 선택 후 **실행**
7. 잡 목록에서 생성된 잡 클릭
8. **실시간 로그** 콘솔에서 `[START] → [AI] → [PLAYWRIGHT] → [DONE]` 흐름 확인
9. 상태가 **통과**로 변경됨 확인

---

## 시나리오 2 — Custom AI 시나리오

**목표**: 자연어 시나리오를 입력하면 AI가 맞춤 스크립트를 생성함을 보여줍니다.

1. 기존 프로젝트 상세 페이지에서 **+ 테스트 실행**
2. 테스트 유형: `Custom - AI 시나리오` 선택
3. 시나리오 설명 입력:
   ```
   메인 페이지에 접속하여 검색창이 존재하는지 확인하고 스크린샷을 찍는다
   ```
4. **실행** 클릭
5. 로그 콘솔에서 `[AI] Generating test script...` 메시지 확인
6. 결과 요약 패널에서 통과/실패 수 및 소요 시간 확인

---

## 시나리오 3 — 실패 케이스 확인

**목표**: 실패 시 상태가 명확히 표시되고 오류 정보가 로그에 남음을 보여줍니다.

1. 새 프로젝트 생성:
   - 이름: `실패 테스트`
   - URL: `http://localhost:9999` (존재하지 않는 주소)
2. Smoke 테스트 실행
3. 로그 콘솔에서 `[ERROR]` 메시지 확인
4. 잡 상태가 **오류** 또는 **실패**로 표시됨 확인

---

## 핵심 강조 포인트

| 포인트 | 설명 |
|---|---|
| **URL → 자동 테스트** | 별도 코드 작성 없이 URL 입력만으로 테스트 실행 |
| **실시간 로그** | WebSocket으로 테스트 진행 상황을 즉시 확인 |
| **AI 스크립트 생성** | 자연어 시나리오 → Playwright 스크립트 자동 변환 |
| **결과 저장** | MinIO에 리포트 저장, 언제든 재확인 가능 |
| **알림** | SNS를 통해 완료/실패 알림 발송 |
